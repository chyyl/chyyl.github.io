一、GitHub Actions 是什么

GitHub Actions 是 GitHub 内置的 CI/CD 平台——你在仓库里放一个 YAML 文件，GitHub 就会在代码推送时自动拉起一台虚拟机，按你写的步骤跑测试、构建、部署。

核心优势：

  • 与 GitHub 仓库零配置集成，不需要额外注册服务
  • 公开仓库完全免费，私有仓库有免费额度
  • 支持 Linux / macOS / Windows / ARM64 四大平台
  • marketplace 上万个现成 Action 可直接复用
  • 2026 年安全路线图加入了依赖锁定、原生出站防火墙等企业级特性

2026 年定价变化

8.1 开启 Step Debug Log

在仓库 Settings → Secrets 添加：

  • ACTIONS_STEP_DEBUG = true

或临时在 URL 加参数：?logLevel=debug

8.2 SSH 进 Runner 调试

运行时会打印 SSH 连接信息，可远程连入 runner 实时调试。

8.3 常见错误排查

问题 原因 解决
Permission denied GITHUB_TOKEN 权限不足 加 permissions: 块
Resource not accessible 同上 检查 contents: write 等
Artifact 上传失败 名称冲突 v4 不允许跨 Job 同名
缓存 miss key 变化 检查 hashFiles 参数
推送 tag 不触发 tags 规则未匹配 检查 on.push.tags
