一、Netlify 是什么

Netlify 是一个面向前端和全栈应用的托管平台。你把代码推到 GitHub，Netlify 自动构建、部署到全球 CDN，给你一个 HTTPS 域名。不需要配 Nginx、不需要买服务器、不需要管证书。

核心能力：

  • 静态站点托管：HTML/CSS/JS 直接部署，全球 CDN
  • 自动构建：Git push 触发构建和部署
  • Serverless Functions：跑后端逻辑（Node.js/Deno/Go/Rust）
  • Edge Functions：在 CDN 边缘节点运行 Deno 代码
  • Forms：无需后端即可接收表单提交
  • 预览部署：每个 PR 一个独立预览 URL
  • 域名 + HTTPS：自动申请和续期 Let’s Encrypt 证书
  • 重定向和重写：netlify.toml 配置路由

二、入门：三分钟上线

2.1 方式一：拖拽部署（最快）

  1. 准备一个含 index.html 的文件夹
  2. 打开 https://app.netlify.com/drop
  3. 把文件夹拖到页面上
  4. 5 秒后得到一个 https://xxx-xxx-xxx.netlify.app 链接

适合快速分享原型或静态页面。

2.2 方式二：Git 持续部署（推荐）

  1. 打开 https://app.netlify.com → “Add new site” → “Import an existing project”
  2. 选择 GitHub/GitLab/Bitbucket
  3. 选仓库，配置构建：
    • Build command: npm run build（或 hugo、npx @11ty/eleventy 等）
    • Publish directory: dist（或 public、_site、build）
  4. 点 Deploy

之后每次 git push，Netlify 自动构建部署。

6.4 Edge vs Serverless 对比

八、域名与 HTTPS

8.1 添加自定义域名

  1. Netlify Dashboard → Domain settings → Add custom domain
  2. 输入域名（如 example.com）
  3. 在域名注册商处修改 DNS：
    • A 记录: 75.2.60.5
    • CNAME: xxx.netlify.app（子域名）
  4. 等待 DNS 生效（几分钟到几小时）
  5. Netlify 自动申请 SSL 证书

8.2 通配符域名

需要 Premium DNS。在 Netlify 添加 *.example.com，配合 Edge Functions 可实现动态子域名路由。

问题 解决方案
构建失败 查看构建日志；检查 Node 版本；本地 netlify build 复现
函数 404 检查 redirect 规则和函数路径
SPA 刷新 404 加 /* → /index.html 回退重写
