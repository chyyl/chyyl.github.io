八、2026 年 Git 生态新变化

8.1 Git 2.46+ 关键特性

  • git switch / git restore：2.23 引入，2026 年已成主流，比 checkout 语义更清晰
  • --format 补全：git log --format=%an 等占位符更丰富
  • Sparse Checkout 改进：--cone 模式成为默认，性能大幅提升
  • git maintenance：后台自动 gc 和 prefetch，仓库不再变慢

GitHub 商业化越来越激进（2026 年 3 月 Actions 定价调整），自托管 Git 平台 Forgejo/Gitea 成为替代选择。迁移成本低，Git 协议通用。

九、常见问题速查

场景 命令
提交错了分支 git reset --soft HEAD~1 → 切到正确分支 → git commit
想撤销 push git revert <commit> → git push
合并冲突太多 git merge --abort 放弃合并
不小心 reset --hard git reflog 找回
拉取代码有冲突 git pull --rebase 保持线性历史
想看某行代码谁写的 git blame <file> -L 10,20
想把文件恢复到某版本 git checkout <commit> -- <file>
删除远程已删的本地分支 git fetch --prune
修改上次提交信息 git commit --amend -m "新信息"
合并多次小提交 git rebase -i HEAD~3 + squash

十、总结

阶段 核心命令 占日常使用
基础 add commit push pull status log 70%
分支 branch switch merge rebase 15%
协作 clone remote fetch cherry-pick 10%
高级 bisect reflog worktree rebase -i 5%

Git 的设计哲学：给你足够的绳子，你可以上吊，也可以绑出花来。 掌握基础三连和分支操作够用大半年；遇到事故翻 reflog；想提效学 rebase -i 和 worktree。
