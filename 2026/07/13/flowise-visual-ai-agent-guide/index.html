拖拽连线就能搭出一个 RAG 聊天机器人、多 Agent 协作系统、带工具调用的智能客服。Flowise 是目前最成熟的开源可视化 AI 应用构建平台，基于 LangChain，自带 REST API，本地部署完全免费。这篇从安装讲起到多 Agent 编排，由浅入深走完一遍。

一、Flowise 是什么

1.1 一句话定义

Flowise 是一个开源的低代码 AI 应用构建平台。你在一个可视化画布上拖拽节点（LLM、向量数据库、文档加载器、记忆模块、工具），用线连起来，就构成了一个可运行的 AI 应用。背后编译成 LangChain 流水线，对外暴露 REST API。

1.2 为什么选 Flowise

对比维度 Flowise Dify n8n LangChain 原生代码
可视化拖拽 ✅ 画布式 ✅ 表单式 ✅ 流程图式 ❌
底层框架 LangChain 自研 自研 LangChain
多 Agent 编排 ✅ LangGraph 集成 ✅ ✅ 需插件 ✅ 手写
RAG 支持 ✅ 完整 ✅ 完整 ⚠️ 基础 ✅ 手写
自带 API ✅ 每个 Chatflow 自动生成 ✅ ✅ Webhook ❌ 需自建
嵌入式聊天 ✅ 一行 JS 嵌入网页 ✅ ❌ ❌
MCP 支持 ✅ Client/Server 节点 ⚠️ ⚠️ ⚠️
本地部署免费 ✅ ✅ 有限制 ✅ ✅
适合人群 开发者+产品 非技术优先 自动化工程师 开发者

Flowise 的最佳场景：

  • 需要 RAG 知识库聊天的客服/文档助手
  • 需要多 Agent 协作的复杂任务流
  • 想快速原型验证 AI 应用想法
  • 需要给非技术人员调整 Prompt 和知识库的能力
  • 需要嵌入式聊天组件挂到网站上

1.3 三种构建模式

Flowise v3 提供三种构建器，难度递增：

Assistant（助手模式） —— 最简单

零配置上手。上传文件就是 RAG，勾选工具就能联网搜索，选个模型就能聊天。适合：快速搭一个知识库问答机器人。

Chatflow（对话流模式） —— 中等灵活度

单 Agent 系统，节点自由组合。可以接 Graph RAG、Reranker、自定义 Retriever、多路召回。适合：需要精细控制 RAG 管线的场景。

Agentflow（智能体流模式） —— 最强大

Chatflow 和 Assistant 的超集，支持多 Agent 协作、条件分支、循环、路由。底层集成 LangGraph。适合：多步骤复杂任务、多角色协作、Agentic RAG。

四、入门：搭一个简单聊天机器人

4.1 创建 Assistant

最快的方式：

  1. 左侧点 Assistants → Add New
  2. 填名字：”我的第一个助手”
  3. 选择模型：ChatOllama → Model 填 qwen3:8b → Base URL 填 http://host.docker.internal:11434
  4. 保存 → 点 Chat 图标 → 开始聊天

就这么简单，你已经有了一个能聊天的 AI 助手。

4.2 创建 Chatflow

更灵活的方式：

  1. 左侧点 Chatflows → Add New → 选 Chatflow

  2. 画布上会自动出现一个基础模板，包含：

    • Chat OpenAI（或 ChatOllama）节点：LLM 模型
    • LLM Chain 节点：将 Prompt 和 LLM 组合
    • Conversational Retrieval QA Chain 节点：带记忆的问答链

  3. 双击 ChatOllama 节点配置：

    • Model: qwen3:8b
    • Base URL: http://host.docker.internal:11434
    • Temperature: 0.7

  4. 保存 → 点击右下角绿色 Chat 图标测试

5.4 步骤三：上传文档创建知识库

方式 A：通过 Flowise 界面上传

  1. 左侧 Knowledge Base → Add New
  2. 拖入 PDF / TXT / Markdown / Word 文件
  3. 选择 Splitter（推荐 Recursive Character Text Splitter）
    • Chunk Size: 1000
    • Chunk Overlap: 200
  4. 选择 Embedding 模型 → 点 Process
  5. 完成后在 Chatflow 中引用这个 Knowledge Base

方式 B：在 Chatflow 中用节点

拖以下节点串联：

支持的 Document Loader：

  • PDF File Loader：加载 PDF
  • Text File Loader：加载 TXT
  • Web Page Loader：输入 URL 抓取网页
  • GitHub Repo Loader：加载 GitHub 仓库文件
  • Notion Loader：从 Notion 拉取
  • CSV Loader：加载表格数据

5.5 步骤四：构建完整 RAG Chatflow

画布上连接以下节点：

配置 Conversational Retrieval QA Chain：

  • LLM: 连接 ChatOllama
  • Vector Store: 连接 Chroma
  • Embeddings: 连接 Ollama Embeddings
  • Memory: 连接 Buffer Memory
  • Return Source Documents: true（让 AI 引用来源）

保存 → 聊天测试：问一个只有文档里才知道的问题，看它是否能正确回答并引用来源。

七、Agentflow：多 Agent 编排

Agentflow 是 Flowise v3 的旗舰功能，基于 LangGraph。

7.1 什么时候用 Agentflow

  • 需要多步骤任务（先搜索 → 再分析 → 再写报告）
  • 需要多角色协作（研究员收集信息 → 写手撰写 → 审校检查）
  • 需要条件分支（简单问题走 SLM，复杂问题走大模型）
  • 需要循环（RAG 结果不好 → 自动改写查询重试）

7.2 构建一个研究 Agent 团队

目标：用户提问 → 研究员搜索资料 → 分析师总结 → 写手出报告

操作步骤：

  1. 左侧 Agentflows → Add New
  2. 拖 Start 节点
  3. 拖三个 LLM Agent 节点，分别命名 Researcher / Analyst / Writer
  4. 每个 Agent 配置不同的 System Prompt 和工具
  5. 用线依次连接
  6. 拖 End 节点
  7. 保存 → 运行测试

7.4 Agentic RAG

传统 RAG 是单次的：查一次 → 答。Agentic RAG 让 Agent 自己判断：

  1. 检索到的内容是否相关？
  2. 不相关 → 改写查询重新检索
  3. 相关 → 生成答案
  4. 答案是否完整？
  5. 不完整 → 补充检索

10.2 Tracing（追踪）

Flowise 集成了 LangSmith / Langfuse 等观测平台：

配置后每次执行都会上报到 LangSmith，可视化查看调用链。

10.3 常见调试技巧

问题 排查方向
模型不回复 检查 Ollama 是否运行、模型是否拉取
RAG 答案不相关 调整 Chunk Size / Overlap、加 Reranker
超时 减小 context size、检查网络
工具不调用 检查工具描述是否清晰、模型是否支持 function calling
内存爆 向量库换持久化存储、限制历史消息长度

十一、Flowise vs 竞品选择指南

你的需求 推荐
快速搭 RAG 聊天机器人 Flowise Assistant 模式
精细控制 RAG 管线 Flowise Chatflow
多 Agent 协作 Flowise Agentflow
非技术人员主导 Dify（表单式更友好）
需要复杂自动化流 n8n + Ollama
需要完全代码控制 LangChain / LangGraph 原生
需要工作流+AI混合 n8n（更偏通用自动化）
需要嵌入式聊天 Flowise（自带嵌入组件）

总结速查

阶段 核心操作 关键节点
安装 Docker Compose 一键起 Ollama + Flowise + PostgreSQL + Redis
入门 Assistant 模式 ChatOllama + 文件上传
RAG Chatflow 模式 Document Loader + Text Splitter + Embeddings + Vector Store
Agent Chatflow + 工具 Tool Calling Agent + Custom Tool
多 Agent Agentflow 模式 LLM Agent + Conditional Edge + Loop
部署 API + 嵌入 REST API + 嵌入式 JS 组件
监控 日志 + Tracing LangSmith / Langfuse
