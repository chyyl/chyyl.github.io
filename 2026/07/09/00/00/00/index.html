8B 参数的模型，量化后只要 2.37GB 内存就能跑。这不是标题党——OpenBMB 的 BitCPM 系列用 1.58 位三值量化（ternary quantization）把模型权重压缩到 {-1, 0, 1} 三个值，比 BF16 减少 90% 的位宽，还能保留 95-97% 的原始性能。这篇文章讲清楚 BitCPM 是什么、1.58 位量化什么原理、怎么用 llama.cpp 在本地跑起来。

一、BitCPM 是什么

1.1 从 MiniCPM 到 BitCPM

OpenBMB 的 MiniCPM 系列大家不陌生——端侧小模型的标杆。BitCPM 是 MiniCPM 的三值量化版本，通过 QAT（量化感知训练）把模型权重从 BF16（16 位浮点）压缩到 1.58 位（三值 {-1, 0, 1}），在保持绝大部分性能的前提下大幅降低内存占用。

BitCPM 目前有两条产品线：

系列 基座 训练方式 可用规格
BitCPM4 MiniCPM3/4 QAT 后量化 0.5B、1B
BitCPM-CANN MiniCPM4 全系列 端到端 1.58-bit 原生训练 0.5B、1B、3B、8B

两条线的区别：

  • BitCPM4：在已有的 MiniCPM 模型上做量化感知微调，属于”后量化”
  • BitCPM-CANN：从零开始在华为昇腾 NPU 上用 1.58 位做完整训练，是全球首个端到端 1.58-bit 训练系统

1.2 BitCPM-CANN 的技术意义

BitCPM-CANN 不只是”又量化了一个模型”，它证明了几件大事：

  1. 三值量化可以从头训练——不需要先训 BF16 再量化，直接在 1.58 位精度下训练，效果相当
  2. 国产 NPU 能搞低比特训练——华为昇腾 910B 上跑通了完整 Megatron-LM + MindSpeed 管线
  3. 训练开销极小——QAT 只引入 5% 的吞吐损失（148 vs 155 TFLOP/s per NPU）
  4. 推理端收益巨大——约 6 倍内存削减

二、1.58 位量化：原理详解

2.1 为什么是 1.58？

三值量化的权重只有三个值：{-1, 0, 1}。

存储一个三值需要一个三进制位（trit），一个 trit 能表示 3 个状态。换算成比特：

所以叫 1.58-bit——每个权重只需要 1.585 个比特来表示。这比 BF16 的 16 位少了 90%。

2.2 实际存储：打包压缩

硬件没法直接存 1.58 个比特，所以实际采用打包策略：

  • 5 个三值权重打包成 5 个 trit
  • 5 个 trit 能表示 3⁵ = 243 个状态
  • 243 < 256 = 2⁸，所以 5 个 trit 刚好装进 8 个比特
  • 实际存储：8/5 = 1.6 bits/weight

这比理论值 1.585 略高，但已经非常接近。llama.cpp 的 TQ2_0 格式用的就是这种打包方式。

2.3 三值 vs 传统量化

维度 BF16（全精度） INT8 量化 INT4 量化 三值（1.58-bit）
位宽 16 bit 8 bit 4 bit ~1.6 bit
权重值 任意浮点 256 级 16 级 3 级（-1/0/1）
压缩比 1× 2× 4× ~10×
性能保留 100% 95-98% 90-95% 95-97%
量化方式 — PTQ/QAT PTQ/QAT QAT（从头训练）
计算特点 浮点乘加 整数乘加 整数乘加 只需加减法

三值量化有个革命性优势：权重是 {-1, 0, 1}，矩阵乘法变成了纯加减法——乘以 -1 是取反，乘以 0 是跳过，乘以 1 是原值。不需要做任何乘法运算。

这意味着：

  • 理论计算速度大幅提升（加法比乘法快得多）
  • 能耗显著降低（乘法器是芯片上最耗电的部件之一）
  • 零权重直接跳过（三分之一的权重是 0，不需要计算）

2.4 为什么性能不掉太多

直觉上，把 16 位压缩到 1.58 位应该损失巨大。但研究表明：

  1. 大模型权重分布近似正态——绝大多数权重值很小，接近 0
  2. 三值化后 {-1, 0, 1} 能捕获主要方向——正负方向 + 不重要
  3. QAT 让模型在训练时就适应低精度——不是事后量化，而是学会了在三个值下工作
  4. 关键层可以保留高精度——如 embedding 层和输出层

BitCPM-CANN 的实测数据：1B/3B/8B 模型在 11 个 benchmark 上保留了 95.7%–97.2% 的全精度性能。0.5B 稍差一些，但也在 90% 以上。

2.5 与其他量化的对比

关键发现：TQ2_0（三值）的体积跟 Q2_K_S 差不多，但性能保留率远超 Q2_K_S——因为三值是训练出来的，不是事后量化的。

8B 模型在 RTX 4060 上跑 60 tok/s——这可是 8B 参数的模型在消费级显卡上的表现。

4.2 三值量化独有优势：跳零加速

三值模型中约三分之一的权重为 0，这些位置的计算可以直接跳过。llama.cpp 对 TQ2_0 格式做了专门的稀疏优化：

  • 0 值跳过：遇到权重 0 直接不计算
  • 符号翻转：权重 -1 只需对激活值取反后相加
  • 无乘法：整个矩阵乘法被替换为加减法序列

这意味着实际计算量远小于等价位宽的传统量化模型。在计算密集型场景下，TQ2_0 的速度可以超过 Q4_0。
